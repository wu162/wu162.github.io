springboot整合mybatis

添加的依赖如下，注意版本号可能不一样没有关系

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>2.1.0</version>
</dependency>
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>druid-spring-boot-starter</artifactId>
    <version>1.1.10</version>
</dependency>
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <scope>runtime</scope>
    <!-- 如果你安装的mysql版本不是新的，比如5.x版本，那么这里要加上你的mysql版本号-->
</dependency>

在 application.properties 里添加数据库相关配置

spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
# 修改为你自己的数据库的用户名和密码
spring.datasource.username=root
spring.datasource.password=123
# 这里ip改为你自己的数据库地址，编码改为你的数据库的编码格式
spring.datasource.url=jdbc:mysql://192.168.66.128:3306/pengion?serverTimezone=UTC&useUnicode=true&characterEncoding=utf-8&useSSL=true

添加实体类，例如

public class User {
    private Integer id;
    private String username;
    private String address;

    //get和set方法，此处省略
}

然后是操作数据库的接口，代码如下

public interface UserMapper {
    //这里仅列出一个方法
    List<User> getAllUser();
}

对应这个接口的sql语句卸载xml里，代码如下

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="org.pengion.mybatis.mapper.UserMapper">
    <!-- 注意id名与方法名一致，namespace与接口文件的位置一致>
    <select id="getAllUser" resultType="org.pengion.mybatis.bean.User">
        select * from user ;
    </select>
</mapper>

还要配置这个xml的位置，不然无法生效，在 application.properties 里添加如下配置

mybatis.mapper-locations=classpath:/mapper/*.xml

还要让程序扫描对应的接口，例如

//MapperScan注解让程序扫描这个包下的所有操作数据库的接口类
@SpringBootApplication
@MapperScan(basePackages = "org.pengion.mybatis.mapper")
public class MybatisApplication {

    public static void main(String[] args) {
        SpringApplication.run(MybatisApplication.class, args);
    }

}

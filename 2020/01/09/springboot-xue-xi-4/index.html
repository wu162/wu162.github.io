springboot和json

springboot导入web模块后默认采用jackson解析json数据，可以添加Gson依赖来使用gson解析数据，如下

<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
</dependency>-->

JSON的生成离不开HttpMessageConverter这个类

全局修改jackson生成的JSON数据的格式

例如想要修改Date类型数据生成的日期格式，有两种方式，

//新增以下类
@Configuration
public class WebMvcConfig {
    //可以添加下面的bean
    @Bean
    MappingJackson2HttpMessageConverter mappingJackson2HttpMessageConverter() {
        MappingJackson2HttpMessageConverter converter = new MappingJackson2HttpMessageConverter();
        ObjectMapper om = new ObjectMapper();
        om.setDateFormat(new SimpleDateFormat("yyyy/MM/dd"));
        converter.setObjectMapper(om);
        return converter;
    }

    //也可以添加这个bean，两种二选一
    @Bean
    ObjectMapper objectMapper() {
        ObjectMapper om = new ObjectMapper();
        om.setDateFormat(new SimpleDateFormat("yyyy/MM/dd"));
        return om;
    }
}

修改Gson生成的JSON数据的格式

若要使用Gson，首先像上面那样添加依赖，然后添加以下类，依然有两种方式

@Configuration
public class WebMvcConfig {
    //可以添加下面的bean
    @Bean
    GsonHttpMessageConverter gsonHttpMessageConverter() {
        GsonHttpMessageConverter converter = new GsonHttpMessageConverter();
        converter.setGson(new GsonBuilder().setDateFormat("yyyy/MM/dd").create());
        return converter;
    }

    //也可以添加这个bean，两种二选一
    @Bean
    Gson gson() {
        return new GsonBuilder().setDateFormat("yyyy/MM/dd").create();
    }
}

修改FastJson生成的JSON数据的格式

也可以使用FastJson解析JSON数据，首先添加如下依赖

<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>fastjson</artifactId>
    <version>1.2.49</version>
</dependency>

然后添加如下类，就可修改FastJson生成的JSON数据的格式

@Configuration
public class WebMvcConfig {
    @Bean
    FastJsonHttpMessageConverter fastJsonHttpMessageConverter() {
        FastJsonHttpMessageConverter converter = new FastJsonHttpMessageConverter();
        FastJsonConfig config = new FastJsonConfig();
        config.setDateFormat("yyyy-MM-dd");
        converter.setFastJsonConfig(config);
        return converter;
    }
}

静态资源路径配置

默认路径有四个地方，优先级从高到低依次为

  • classpath: /MBTA-INF/resources/
  • classpath: /resources/
  • classpath: /static/
  • classpath: /public/

可以自己配置静态资源的路径，在 application.properties 文件里添加以下配置

spring.resources.static-locations=classpath:/pengion/      //此处改为你定义的路径

文件上传

单文件上传

代码如下

@RestController
public class FileUploadController {

    SimpleDateFormat sdf = new SimpleDateFormat("/yyyy/MM/dd/");

    @PostMapping("/upload")
    public String upload(MultipartFile file, HttpServletRequest req) {
        String format = sdf.format(new Date());
        //将图片按日期归档
        String realPath = req.getServletContext().getRealPath("/img") + format;
        File folder = new File(realPath);
        if (!folder.exists()) {
            folder.mkdirs();
        }
        //原来的图片名字
        String oldName = file.getOriginalFilename();
        //名字加上随机id
        String newName = UUID.randomUUID().toString() + oldName.substring(oldName.lastIndexOf("."));
        try {
            file.transferTo(new File(folder, newName));
            String url = req.getScheme() + "://" + req.getServerName() + ":" + req.getServerPort() + "/img" + format + newName;
            return url;
        } catch (IOException e) {
            e.printStackTrace();
        }
        return "error";
    }
}

对应的前端页面如下

    <input type="submit" value="提交">
</form>

</html>

多文件上传

代码如下

@RestController
public class FileUploadController {

    SimpleDateFormat sdf = new SimpleDateFormat("/yyyy/MM/dd/");

    @PostMapping("/uploads")
    public String uploads(MultipartFile[] files, HttpServletRequest req) {
        String format = sdf.format(new Date());
        String realPath = req.getServletContext().getRealPath("/img") + format;
        File folder = new File(realPath);
        if (!folder.exists()) {
            folder.mkdirs();
        }
        //注意此时文件是一个数组
        for (MultipartFile file : files) {
            String oldName = file.getOriginalFilename();
            String newName = UUID.randomUUID().toString() + oldName.substring(oldName.lastIndexOf("."));
            try {
                file.transferTo(new File(folder, newName));
                String url = req.getScheme() + "://" + req.getServerName() + ":" + req.getServerPort() + "/img" + format + newName;
                System.out.println(url);
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        return "success";
    }
}

对应的前端页面如下

<!DOCTYPE html>

</head>

<form action="/uploads" method="post" enctype="multipart/form-data">
    <input type="file" name="files" multiple>
    <input type="submit" value="提交">
</form>
</body>
</html>

@ControllerAdvice的两种用法

处理全局异常

代码如下，用到了web依赖和Thymeleaf依赖

@ControllerAdvice
public class MyCustomException {
    //指明要处理的异常
    @ExceptionHandler(MaxUploadSizeExceededException.class)
    public ModelAndView myexception(MaxUploadSizeExceededException e) throws IOException      {
        ModelAndView mv = new ModelAndView("myerror");
        mv.addObject("error", "上传文件大小超出限制！");
        return mv;
    }
}

对应的模板页面为

<html lang="en" xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
<h1 th:text="${error}"></h1>
</body>
</html>

获取全局数据的方式如下

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello(Model model) {
        Map<String, Object> map = model.asMap();
        Set<String> keySet = map.keySet();
        for (String key : keySet) {
            System.out.println(key + ":" + map.get(key));
        }
        return "hello";
    }
}

自定义错误页

可以对404,500等这些错误定制错误页面

如果是静态页面，以对应的状态码命名html文件放到静态资源路径下，发生对应的错误时就会调用对应的页面，例如在 classpath:/resources/static/ 下新建 404.html 页面，那么发生404错误时就会调用这个页面

也可以新建 4xx.html 页面匹配所有以4开头的错误对应的错误页面

如果是动态页面，同理在 classpath:/resources/templates/ 下以对应的状态码命名html页面，例如404.html

错误页面的匹配规则：精确>模糊，动态>静态 。例如先找404.html，然后4xx.html，先匹配动态页面，再匹配静态页面

注册拦截器

拦截器的代码如下

//自定义拦截器
public class MyInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("preHandle");
        return true;
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("postHandle");
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("afterCompletion");
    }
}

注册拦截器的代码如下

@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(myInterceptor()).addPathPatterns("/**"); //匹配所有路径
    }
    @Bean
    MyInterceptor myInterceptor() {
        return new MyInterceptor();
    }
}

ApplicationRunner实现系统启动任务

代码如下

//，有多个启动任务时，Order注解指定顺序，数字越小优先级越高
@Component
@Order(99)
public class MyApplicationRunner01 implements ApplicationRunner {
    @Override
    public void run(ApplicationArguments args) throws Exception {
        String[] sourceArgs = args.getSourceArgs();//获取启动的所有参数
        System.out.println("sourceArgs:" + Arrays.toString(sourceArgs));
        List<String> nonOptionArgs = args.getNonOptionArgs();
        System.out.println("nonOptionArgs:" + nonOptionArgs);
        Set<String> optionNames = args.getOptionNames();
        System.out.println(">>>>>>>>>>>>>>>>>>>>>>>>>");
        for (String optionName : optionNames) {
            System.out.println(optionName + ":" + args.getOptionValues(optionName));
        }
        System.out.println(">>>>>>>>>>>>>>>MyApplicationRunner01结束>>>>>>>>>>>>>>>>");
    }
}

三种参数的区别：如果启动参数为 --name=pengion --address=wu162.github.io 三国演义 红楼梦

输出的结果为

sourceArgs:[--name=pengion, --address=wu162.github.io, 三国演义, 红楼梦]
nonOptionArgs:[三国演义, 红楼梦]
>>>>>>>>>>>>>>>>>>>>>>>>>
address:[wu162.github.io]
name:[pengion]
>>>>>>>>>>>>>>>MyApplicationRunner01结束>>>>>>>>>>>>>>>>
